User manual - V1


Getting Started

Our API is accessible as a web service. You can access our endpoint here at:

http://api.kaiworship.xyz/v1/

To use our API, append desired parameters to the end of the above URL in the format described in API Calling Convention. You can then make a HTTP request using this URL.

Alternatively, you can use our tool to generate a URL based on your desired parameters. The tool also displays the response our API would provide.



Functionality

Data sourced directly from the Australian Bureau of Statistics (ABS) is cluttered and hard to read. This API provides ABS data and statistics on the Retail industry and on Merchandise Exports in a human readable format. The core functionality of the API is outlined below:

Users are able to specify whether they want to obtain Retail industry data, or Merchandise Exports data.

For Retail industry data, users are able to:
  1. Obtain data on one or more retail industry categories, including the total for all categories.
  2. Obtain data on one or more Australian states/territories, including the total for the country.
  3. Specify dates between which the data will be obtained.
  4. For each retail category, in each state, users are returned with the following information:
    1. Retail turnover (in millions of $A)
    2. The month this data is for
For Merchandise Export data, users are able to:
  1. Obtain data on one or more export commodity, including the total for all exports.
  2. Obtain data on one or more Australian states/territories, including the total for the country.
  3. Specify dates between which the data will be obtained.
  4. For each export commodity, in each state, users are returned with the following information:
    1. Value of exports (in thousands of $A)
    2. The month this data is for


API Calling Convention

Parameters will be passed to our module through a HTTP GET call with the following parameters, and be templated in the following way:

http://api.kaiworship.xyz/v1/{statisticsArea}/{category}/{state}?startDate={startDate}&endDate={endDate}



Parameters
Parameter Arguments and Formats Requirement Separated By Example
statisticsArea A string which can take value “Retail” or “MerchandiseExports” Mandatory N/A “Retail”
category A list of one or more categories, separated by commas, in the form of a string. Mandatory Comma “Food,HouseholdGood”
state A list of one or more states/territories, separated by commas, in the form of a string. If no paramaters are defined 'Total' will be set as default. Optional Comma “SA,VIC,NT”
startDate A date string with the format “YYYY-MM-DD”. Optional N/A “2012-12-12”
endDate A date string with the format “YYYY-MM-DD” Optional N/A “2017-01-27”

Due to the start and end data being optional, there are various cases
  • If no start date and no end date provded. Data will be provided for previous 12 months from today
  • If only start date is provided. The end date will be set to a year after the start date
  • If only end data is provided. The Start date will be set to a year before the end date


    • Acceptable Input

    Below are all the possible values which can be used as parameters in a call to our API. Please only use values listed in the table below.

    Parameter Remarks Available Options
    StatisticsArea
    1. Retail
    2. Merchandise
    Category (retail) If Statistics Area is chosen to be Retail, you are able to choose one or more of the following Industries.
    1. Total
    2. Food
    3. HouseholdGood
    4. ClothingFootwareAndPersonalAccessory
    5. DepartmentStores
    6. CafesRestaurantsAndTakeawayFood
    7. Other
    Commodities (Merchendise Exports) If Statistics Area is chosen to be Merchandise Exports, you are able to choose one or more of the following Commodities
    1. Total
    2. FoodAndLiveAnimals
    3. BeveragesAndTobacco
    4. CrudeMaterialAndInedible
    5. MineralFuelLubricantAndRelatedMaterial
    6. AnimalAndVegetableOilFatAndWaxes
    7. ChemicalsAndRelatedProducts
    8. ManufacturedGoods
    9. MachineryAndTransportEquipments
    10. OtherManufacturedArticles
    11. Unclassified
    States You are able to choose one or multiple values from the states. Use 'Total' for statistics on the entire country.
    1. Total
    2. NSW
    3. WA
    4. SA
    5. ACT
    6. VIC
    7. TAS
    8. QLD
    9. NT


    How to Integrate our API within a Webapp

    Here are examples of how to call this API in various languages, assuming the url is defined as a variable url

    Python:
    
              import urllib2
              response = urllib2.urlopen(url)
              json_string = response.read()
    JavaScript (ES6, Experiment version, supported by latest Chrome and Node.js Engine):
    
            fetch(url)
                .then((data) => {
                    // handle data
                    console.log(data);
                })
                .catch((error) => {
                    //handle error
                });


    API response

    Our API returns a response to a call in JSON format. The following is an outline of the response's JSON properties:

    Property Description
    Statistics Area The statistics area chosen to be queried, that is:
  • "MonthlyRetailData" for Retail
  • "MonthlyCommodityExportData" for Merchandise Exports
    		•
    Category The industry-specific category as listed above in the Acceptable Input section.
    States States of Australia as chosen by the user, in their 3-letter abbreviation
    1. Australia - AUS
    2. New South Wales - NSW
    3. Western Australia - WA
    4. South Australia - SA
    5. Australian Captial Territory - ACT
    6. Victoria - VIC
    7. Tasmania - TAS
    8. Queensland - QLD
    9. Northern Territory - NT
    Data The data for the specified month, state and category. This contains within it:

    Date - the last date of the month in which the data pertains to (in the format YYYY-MM-DD)
    Value - the actual data value, which is:
  • for Retail: Turnover in millions of $A
  • for MerchandiseExports: Export value in thousands of $A
    		•
