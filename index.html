User manual

Functionalities
  • Ability to get Merchandise and Retail data of one or more certain states within the certain category
  • Supporting both in browser viewing and external API calls.
API calling convention

Parameters will be passed to our module through a REST GET call with the following parameters, and be templated in the following way:

http://api.kaiworship.xyz/v2/{statisticsArea}/{category}/{state}?startDate={startDate}&endDate={endDate}
Parameter Arguments and Formats Requirement Separated By Example
statisticsArea A string which can take value “Retail” or “MerchandiseExports” Mandatory N/A “Retail”
category A list of one or more categories, separated by commas, in the form of a string. Mandatory Comma “Food,HousholdGood”
state A list of one or more states/territories, separated by commas, in the form of a string. Optional Comma “SA,VIC,NT”
startDate A date string with the format “YYYY-MM-DD” Optional N/A “2012-12-12”
endDate A date string with the format “YYYY-MM-DD” Optional N/A “2017-01-27”
